User's guide

The purpose of Threeangular is to avoid boilerplate related to creating Three.js pages.
It is not a framework, I really like Three.js and I'm sure that it has to have all the options it has, but still sometimes, specially for new users I think WebGL and Three.js can be overcomplicated and a daunting task to get it running.

Nevertheless it will be necessary to create some files for it to be integrated with Angular JS

  • ngmodule.js Its necessary for Angular to run.
  • ngcontroller.js Contains the controller in which resides our code. It could be included in the main js file but as the project growths bigger it would be easier to update in a separate file.
